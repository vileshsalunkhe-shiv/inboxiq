## Common Causes of "invalid_grant"

### 1. **Redirect URI Mismatch** ⭐ Most Likely
Google is EXTREMELY strict about redirect_uri matching.

**Symptom:** Error happens immediately when exchanging code  
**Cause:** The `redirect_uri` in token exchange doesn't EXACTLY match authorization URL  
**Fix:** Ensure both use identical URI (including trailing slash, http vs https, port, etc.)

**Current URIs:**
- **Authorization:** Generated by `AuthService.build_google_auth_url()`
- **Token Exchange:** Uses `settings.google_redirect_uri`
- **Railway Production:** `https://inboxiq-production-5368.up.railway.app/auth/google/callback`
- **Local Development:** `http://localhost:8000/auth/google/callback`

**Debug Steps:**
1. Log the exact `redirect_uri` used in authorization URL
2. Log the exact `redirect_uri` used in token exchange
3. Compare character-by-character
4. Check for:
   - Trailing slash differences (`/callback` vs `/callback/`)
   - Protocol mismatch (`http://` vs `https://`)
   - Port differences (`localhost:8000` vs `localhost`)
   - Extra query parameters

---

### 2. **Authorization Code Expired**
Google auth codes expire in 10 minutes (sometimes less).

**Symptom:** Error happens if user delays after auth  
**Cause:** Too much time between authorization and code exchange  
**Fix:** Exchange code immediately (already doing this)

**Current Implementation:**
- iOS gets code from callback → immediately sends to backend
- Backend immediately exchanges code for tokens
- **Timing should be <1 second**

**Debug Steps:**
1. Add timestamp logging to track code age
2. Measure time from authorization → callback → token exchange

---

### 3. **Code Already Used**
Auth codes are single-use only.

**Symptom:** Error on second attempt with same code  
**Cause:** Retry logic using same code  
**Fix:** Never retry with same code

**Current Implementation:**
- No retry logic visible in code
- Each auth flow generates new code
- **This is unlikely the issue**

**Debug Steps:**
1. Check for any retry/duplicate request logic
2. Add unique request ID to track attempts

---

### 4. **Client ID/Secret Mismatch**
Wrong credentials for the Google Cloud project.

**Symptom:** Consistent error regardless of code freshness  
**Cause:** Using credentials from different GCP project  
**Fix:** Verify credentials match the project that generated the code

**Current Implementation:**
- Backend uses `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` from env vars
- Railway deployment has these as environment variables

**Debug Steps:**
1. Verify GCP project OAuth credentials
2. Check that iOS uses same OAuth client
3. Confirm redirect URIs are registered in GCP Console

---

### 5. **Railway Proxy Issues**
Railway might modify requests/headers in unexpected ways.

**Symptom:** Works locally, fails on Railway  
**Cause:** Railway reverse proxy interfering  
**Fix:** Test locally to isolate Railway-specific issues

**Debug Steps:**
1. Run backend locally on `localhost:8000`
2. Point iOS to local backend (or use ngrok)
3. Test OAuth flow
4. If local works → Railway config issue
5. If local fails → OAuth implementation issue

### Phase 2: Add Enhanced Logging

**Goal:** Get detailed diagnostic information

**Changes to `auth_service.py`:**

```python
async def exchange_code_for_tokens(self, code: str, redirect_uri: Optional[str] = None) -> dict:
    """Exchange authorization code for Google OAuth tokens."""
    redirect = redirect_uri or settings.google_redirect_uri
    
    # ADDED: Enhanced logging
    import structlog
    logger = structlog.get_logger()
    
    data = {
        "code": code,
        "client_id": settings.google_client_id,
        "client_secret": settings.google_client_secret,
        "redirect_uri": redirect,
        "grant_type": "authorization_code",
    }
    
    # Log request details (redact secrets)
    logger.info(
        "google_oauth_token_exchange_attempt",
        redirect_uri=redirect,
        client_id_prefix=settings.google_client_id[:20],
        code_length=len(code),
        code_prefix=code[:20],
    )
    
    async with httpx.AsyncClient(timeout=15) as client:
        response = await client.post(GOOGLE_OAUTH_TOKEN_URL, data=data)
        
        # Log response
        logger.info(
            "google_oauth_token_exchange_response",
            status_code=response.status_code,
            response_preview=response.text[:200],
        )
        
        if response.status_code != 200:
            # Detailed error logging (ALREADY EXISTS)
            error_detail = response.text
            logger.error(
                "google_oauth_token_exchange_failed",
                status_code=response.status_code,
                error_detail=error_detail,
                client_id_prefix=data['client_id'][:20],
                redirect_uri=data['redirect_uri'],
                code_prefix=data['code'][:20] if data['code'] else None,
            )
        
        response.raise_for_status()
        return response.json()
```

   ```
   http://localhost:8000/auth/google/callback

   ```
5. **Check Application type:** Web application (not iOS or other)
6. **Note the Client ID** - must match `GOOGLE_CLIENT_ID` env var

**Common Mistakes:**
- ❌ Using iOS OAuth client for backend
- ❌ Missing local or production redirect URI
- ❌ Typo in redirect URI (extra `/`, wrong protocol)
- ❌ Client ID doesn't match env var

### Phase 5: Temporary Test Token Workaround (INB-3)

**Goal:** Unblock testing while debugging OAuth

**Implementation:**

Add endpoint to accept manually obtained token:

```python
@router.post("/auth/test-token")
async def test_token_auth(
    access_token: str,
    refresh_token: str,
    db: AsyncSession = Depends(get_db)
) -> TokenPair:
    """FOR TESTING ONLY: Manually store Google tokens and issue JWTs."""
    if settings.environment != "development":
        raise HTTPException(status_code=403, detail="Only available in development")
    
    # Get user info from Google
    async with httpx.AsyncClient(timeout=10) as client:
        response = await client.get(
            "https://www.googleapis.com/oauth2/v2/userinfo",
            headers={"Authorization": f"Bearer {access_token}"},
        )
        response.raise_for_status()
        profile = response.json()
    
    email = profile.get("email")
    if not email:
        raise HTTPException(status_code=400, detail="Missing email")
    
    # Find or create user
    result = await db.execute(select(User).where(User.email == email))
    user = result.scalar_one_or_none()
    if not user:
        user = User(email=email)
        db.add(user)
        await db.commit()
        await db.refresh(user)
    
    # Store tokens
    auth_service = AuthService(db)
    await auth_service.store_google_tokens(user, {"refresh_token": refresh_token})
    
    # Issue JWTs
    access_jwt, refresh_jwt, expires_in = await auth_service.create_token_pair(str(user.id))
    return TokenPair(access_token=access_jwt, refresh_token=refresh_jwt, expires_in=expires_in)
```

**Usage:**
1. Get tokens from OAuth Playground (Phase 4)
2. POST to `/auth/test-token` with tokens
3. Use returned JWTs to test email sync, AI categorization

**Benefits:**
- ✅ Unblocks testing of core features
- ✅ Can validate email sync works
- ✅ Can test AI categorization
- ✅ Parallel work while debugging OAuth
